# `search_dinov2_topk.py` 분석

## 1. 개요

이 스크립트는 입력 이미지 한 장을 DINOv2 ViT-S/14 임베딩으로 변환한 뒤, `build_dinov2_index.py`가 생성한 인덱스에서 코사인 유사도가 높은 알약 이미지를 Top-K 방식으로 검색한다.

검색 결과에는 유사도, 품목일련번호, 제형, 이미지 면 구분, 품목명과 이미지 경로가 표시된다. 기본 실행 장치는 CPU이며, CUDA와 Intel XPU는 명령행 옵션으로 명시해야 한다.

## 2. 기본 파일

기본 경로는 스크립트가 있는 디렉터리를 기준으로 한다.

| 구분 | 기본값 | 용도 |
|---|---|---|
| 모델 가중치 | `dinov2_vits14_pretrain.pth` | 검색 이미지의 임베딩 추출 |
| 검색 인덱스 | `dinov2_pill_index.pt` | 비교 대상 임베딩과 메타데이터 |

검색할 쿼리 이미지 경로는 위치 인자로 반드시 전달해야 한다.

## 3. 명령행 옵션

| 인자/옵션 | 형식/선택값 | 기본값 | 설명 |
|---|---|---|---|
| `query` | 경로 | 필수 | 검색할 이미지 파일 |
| `--index` | 경로 | `dinov2_pill_index.pt` | 검색 인덱스 파일 |
| `--weights` | 경로 | `dinov2_vits14_pretrain.pth` | DINOv2 가중치 파일 |
| `--top-k` | 정수 | `10` | 출력할 최대 결과 수. 1 이상이어야 함 |
| `--device` | `cpu`, `cuda`, `xpu` | `cpu` | 모델 추론 장치 |
| `--unique-item` | 플래그 | 사용 안 함 | 같은 품목일련번호에서 최고 점수 이미지 하나만 출력 |

CPU 실행 예시:

```powershell
python .\search_dinov2_topk.py .\query.jpg
```

CUDA 또는 XPU 실행 예시:

```powershell
python .\search_dinov2_topk.py .\query.jpg --device cuda --top-k 20
python .\search_dinov2_topk.py .\query.jpg --device xpu --top-k 20
```

품목일련번호별로 하나의 결과만 출력하려면 다음과 같이 실행한다.

```powershell
python .\search_dinov2_topk.py .\query.jpg --unique-item
```

## 4. 전체 처리 흐름

1. 명령행 인자를 파싱하고 `top-k` 값이 1 이상인지 확인한다.
2. 쿼리 이미지, 인덱스, 모델 가중치 파일의 존재 여부를 검사한다.
3. 인덱스를 CPU 메모리로 불러온다.
4. 인덱스의 모델명과 임베딩·레코드 개수를 검증한다.
5. CPU, CUDA 또는 XPU 장치를 선택한다.
6. DINOv2 ViT-S/14 모델 구조와 로컬 가중치를 불러온다.
7. 쿼리 이미지를 전처리하고 L2 정규화된 임베딩을 추출한다.
8. 인덱스 임베딩과 쿼리 임베딩의 내적으로 코사인 유사도를 계산한다.
9. 유사도가 높은 순서로 정렬한다.
10. 쿼리 이미지 자체와 필요시 중복 품목을 제외하고 Top-K 결과를 출력한다.

## 5. 주요 함수

### `parse_args()`

명령행 인자를 정의한다. `auto` 장치 선택은 제공하지 않으며 `--device`를 생략하면 항상 CPU를 사용한다. CUDA와 XPU를 사용하려면 각각 `--device cuda`, `--device xpu`를 지정해야 한다.

### `is_xpu_available()`

현재 PyTorch에 `torch.xpu` 속성이 있고 Intel XPU를 사용할 수 있는지 검사한다. XPU 지원 PyTorch가 아니거나 장치가 준비되지 않은 환경에서도 속성 접근 오류가 발생하지 않도록 먼저 `hasattr()`를 사용한다.

### `select_device(requested)`

- `cpu`: 별도 가용성 검사 없이 CPU 장치를 반환한다.
- `cuda`: `torch.cuda.is_available()`이 거짓이면 오류를 발생시킨다.
- `xpu`: `torch.xpu`가 없거나 `torch.xpu.is_available()`이 거짓이면 XPU 지원 PyTorch와 Intel 그래픽 드라이버를 확인하라는 오류를 발생시킨다.

### `load_checkpoint(path)`

체크포인트를 CPU로 읽는다. 최신 PyTorch에서는 `weights_only=True`를 사용하고, 이 인자를 지원하지 않는 이전 버전에서는 일반 `torch.load()`로 재시도한다.

최상위 딕셔너리에 `model` 키가 있으면 그 값을 state dict로 사용한다. 최종 값이 딕셔너리가 아니면 지원하지 않는 체크포인트 형식으로 처리한다.

### `load_model(weights, device)`

PyTorch Hub에서 다음 DINOv2 모델 구조를 가져온다.

```python
torch.hub.load(
    "facebookresearch/dinov2",
    "dinov2_vits14",
    pretrained=False,
    trust_repo=True,
)
```

실제 가중치는 로컬 체크포인트에서 `strict=True`로 적용한다. 모델과 체크포인트 키가 정확히 일치해야 하며, 모델은 평가 모드로 전환된 후 선택한 CPU, CUDA 또는 XPU 장치로 이동한다.

PyTorch Hub 캐시에 DINOv2 소스가 없으면 최초 실행 시 네트워크 연결이 필요할 수 있다.

### `extract_query_embedding(model, query_path, device)`

쿼리 이미지에 인덱스 생성 때와 동일한 전처리를 적용한다.

1. RGB 변환
2. Bicubic 보간을 사용하는 `Resize(256)`
3. 중앙 `224 × 224` 자르기
4. 텐서 변환
5. ImageNet 평균과 표준편차로 정규화
6. 배치 차원 추가
7. 선택한 장치로 이동

정규화 값은 다음과 같다.

```text
mean = (0.485, 0.456, 0.406)
std  = (0.229, 0.224, 0.225)
```

모델 추론은 `torch.inference_mode()`에서 수행한다. 출력은 `float32`로 변환하고 L2 정규화한 뒤 CPU로 이동하며, 배치 차원을 제거해 384차원 벡터 하나를 반환한다.

### `safe_torch_load(path)`

인덱스에는 텐서뿐 아니라 레코드 리스트와 문자열 등이 포함되므로 `weights_only=False`로 읽는다. 해당 인자를 지원하지 않는 PyTorch 버전에서는 일반 `torch.load()`로 재시도한다.

신뢰할 수 없는 `.pt` 파일을 일반 `torch.load()`로 읽으면 임의 코드 실행 위험이 있으므로, 직접 생성했거나 출처를 신뢰할 수 있는 인덱스만 사용해야 한다.

## 6. 인덱스 검증과 호환 조건

불러온 인덱스에서 다음을 확인한다.

- `model_name`이 정확히 `dinov2_vits14`인지 검사한다.
- `embeddings`를 `float32`로 변환한다.
- 임베딩 행 개수와 `records` 원소 개수가 같은지 검사한다.

다만 다음 항목은 별도로 검증하지 않는다.

- `format_version`
- `embedding_dim`
- `embeddings`와 쿼리 벡터의 실제 차원
- 각 레코드에 필요한 키가 모두 존재하는지 여부
- 검색에 사용하는 가중치가 인덱스 생성 때 사용한 가중치와 동일한지 여부

인덱스와 검색 모델의 가중치가 다르면 같은 모델 구조라도 검색 품질이 저하되거나 임베딩 공간이 호환되지 않을 수 있다.

## 7. 유사도 계산

인덱스와 쿼리의 임베딩은 모두 L2 정규화되었다는 전제로 다음 행렬·벡터 곱을 수행한다.

```python
similarities = embeddings @ query
```

단위 벡터의 내적은 코사인 유사도와 같으므로 결과값이 클수록 시각적으로 유사하다고 판단한다. 모든 점수를 내림차순 정렬한 후 앞에서부터 조건을 만족하는 결과를 선택한다.

인덱스 임베딩과 유사도 계산은 CPU에서 수행된다. `--device cuda` 또는 `--device xpu`는 현재 쿼리 이미지의 모델 추론에만 적용되며, 전체 인덱스는 가속 장치로 이동하지 않는다. 따라서 인덱스가 매우 큰 경우 검색 정렬과 내적 성능은 CPU 성능의 영향을 받는다.

## 8. 결과 필터링

### 쿼리 이미지 자체 제외

레코드의 이미지 절대 경로와 쿼리 이미지의 절대 경로가 같으면 결과에서 제외한다. 경로가 다르지만 내용이 동일한 복사본은 제외되지 않는다.

### `--unique-item`

옵션을 사용하면 이미 결과에 포함된 `item_id`를 집합으로 추적하여 같은 품목일련번호의 후속 이미지를 건너뛴다. 정렬 순서상 가장 높은 점수의 이미지 하나만 남는다.

옵션을 사용하지 않으면 동일 품목의 앞면·뒷면 이미지 등이 각각 검색 결과에 포함될 수 있다.

필터링 때문에 조건을 만족하는 후보 수가 부족하면 요청한 `top-k`보다 적은 결과가 출력될 수 있다.

## 9. 출력 형식

먼저 쿼리 이미지의 절대 경로와 실제 결과 개수를 출력한다. 각 결과에는 다음 값이 포함된다.

```text
 1. similarity=0.912345  item_id=품목일련번호  shape=제형  side=면 구분
    품목명: 제품명
    이미지: 이미지 절대 경로
```

유사도는 소수점 아래 여섯 자리까지 표시한다.

## 10. 오류 조건

다음 상황에서는 실행을 중단한다.

- `top-k`가 1보다 작음
- 쿼리 이미지 파일이 없음
- 인덱스 파일이 없음
- 가중치 파일이 없음
- `--device cuda`를 지정했지만 CUDA를 사용할 수 없음
- `--device xpu`를 지정했지만 Intel XPU를 사용할 수 없음
- 체크포인트 형식이 지원되지 않음
- 체크포인트와 모델 구조가 일치하지 않음
- 인덱스의 `model_name`이 `dinov2_vits14`가 아님
- 인덱스의 임베딩 수와 레코드 수가 다름
- 이미지 파일을 PIL로 열거나 처리할 수 없음
- 인덱스에 필수 키가 없거나 임베딩 차원이 맞지 않음

마지막 두 종류는 별도 설명을 붙여 검사하지 않기 때문에 라이브러리의 원래 예외가 그대로 발생할 수 있다.

## 11. 주의점

- 인덱스 생성과 검색에서 같은 DINOv2 가중치 및 이미지 전처리를 사용해야 한다.
- 중앙 자르기를 사용하므로 쿼리의 알약이 화면 중앙에 위치하는 것이 좋다.
- XPU 사용에는 XPU 지원 PyTorch와 호환되는 Intel 그래픽 드라이버가 필요하다.
- 모델 구조를 Hub 캐시에서 찾을 수 없으면 네트워크가 필요하다.
- 전체 후보를 `torch.argsort()`로 정렬하므로 후보가 매우 많을 때는 `torch.topk()`보다 불필요한 정렬 비용이 발생할 수 있다.
- 쿼리 추론은 GPU/XPU에서 실행할 수 있지만 검색 인덱스 연산은 CPU에서 수행된다.
- `.pt` 인덱스는 신뢰할 수 있는 파일만 불러와야 한다.

## 12. 요약

이 스크립트는 쿼리 이미지의 DINOv2 임베딩과 기존 알약 이미지 임베딩의 코사인 유사도를 비교해 가장 비슷한 이미지를 출력한다. 기본 장치는 CPU이고 CUDA와 XPU는 명시적으로 선택해야 한다. `--unique-item`을 사용하면 품목일련번호별 최고 점수 결과만 남길 수 있다.
