# 알약 제형 유사도 검색 PoC 평가

## 1. 평가 목적

본 PoC는 클래스별 이미지가 매우 적은 환경에서 딥러닝 분류 모델을 직접 학습하는 대신, 알약 윤곽에서 수작업 형상 임베딩을 추출하고 프로토타입과의 거리를 비교하여 의약품제형을 추측할 수 있는지 검토하기 위해 수행했다.

이번 결과는 29개 제형을 완전히 자동 분류하는 최종 모델의 성능을 입증하기 위한 것이 아니라, 소수 표본 환경에서 형상 기반 후보 검색이 가능한지 확인하고 후속 개발에 필요한 제약 조건을 파악하기 위한 기술 타당성 검증이다.

## 2. 구현 내용

다음 특징을 조합하여 56차원 형상 임베딩을 구성했다.

- 가로세로비, 원형도, 타원 이심률
- 직사각형 충전율, 볼록성, 오목도
- 두 축의 대칭도
- Elliptic Fourier Descriptor(EFD)
- 중심에서 윤곽까지의 방사형 거리 FFT
- Hu moment

이미지별 특징을 품목별 중앙값으로 집계하고, 다시 제형별 중심 프로토타입을 생성했다. 하나의 제형 중심만으로 클래스 내부의 다양한 변형을 충분히 표현하기 어려운 문제를 보완하기 위해 품목별 대표 벡터도 exemplar prototype으로 저장했다.

추론 단계에서는 이미지에서 하나 이상의 알약 영역을 검출하고, 각 제형에 속한 품목 프로토타입까지의 거리를 비교하여 Top-k 후보를 반환한다. 앞면과 뒷면이 한 이미지에 함께 있는 경우에는 각 객체의 결과와 이를 종합한 최종 결과를 제공한다.

## 3. 데이터 처리 결과

- 대상 제형: 29개
- 인덱싱 품목: 27,345개
- 특징 추출 성공 이미지: 54,282장
- 특징 추출 실패 이미지: 1장
- 제형 충돌로 제외한 품목: 2개
- 제형 충돌로 제외한 이미지: 18장

제형 정보가 충돌하는 품목은 임베딩 생성에서 제외했다. 원본 이미지나 CSV 데이터는 변경하지 않았다.

## 4. 확인된 성과

### 4.1 소수 데이터 환경에서의 자동 처리 기반 구축

별도의 딥러닝 학습 없이 5만 장 이상의 이미지에서 윤곽 특징을 자동 추출하고, 2만 7천여 품목을 검색할 수 있는 형상 인덱스를 구축했다. 클래스당 이미지가 매우 적더라도 특징 계산 자체는 가능하므로, 데이터 부족으로 학습이 불가능한 제형도 검색 대상에 포함할 수 있다.

### 4.2 형상 중심의 비교 가능

색상, 각인 문자, 제품명보다 외곽 윤곽에 집중하여 원형, 타원형, 장방형, 사과형, 땅콩형 등의 유사도를 비교할 수 있다. 회전과 반전에 따른 특징 변화를 줄이기 위한 정규화도 적용했다.

### 4.3 후보군 축소 가능성 확인

테스트 이미지에서는 장방형, 사과형, 땅콩형 등의 CSV 제형이 Top-1으로 확인됐다. 이를 통해 형상 임베딩이 가능한 제형 후보를 Top-k로 축소하는 보조 검색 수단으로 활용될 가능성을 확인했다.

단, 소수의 테스트 사례가 전체 데이터에 대한 일반적인 정확도를 의미하지는 않는다. 별도의 품목 단위 검증셋 평가가 필요하다.

### 4.4 판정 근거 제공

결과에는 다음 정보가 포함된다.

- 제형별 거리와 상대 점수
- Top-1과 Top-2의 거리 차이
- 가장 유사한 실제 학습 품목
- 제형 중심 프로토타입까지의 거리
- 객체별 판정과 다중 객체 종합 판정

따라서 단순 클래스명만 반환하는 모델보다 결과를 검토하고 오류 원인을 분석하기 쉽다.

## 5. 확인된 한계

### 5.1 클래스 간 형상 중첩

원형과 타원형, 타원형과 장방형, 땅콩형과 8자형처럼 형태 경계가 연속적인 제형은 임베딩 공간에서도 분포가 중첩된다. 이 경우 Top-1과 Top-2의 거리가 매우 비슷하여 하나의 제형으로 확정하기 어렵다.

### 5.2 희소 클래스의 대표성 부족

일부 제형은 품목과 이미지가 6개 이하이므로 클래스의 가능한 변형을 충분히 반영하기 어렵다. 소수 샘플의 촬영 각도나 윤곽 추출 오류가 해당 제형 전체의 대표 벡터에 큰 영향을 줄 수 있다.

### 5.3 클래스 내부 변이

같은 제형명 안에서도 가로세로비, 모서리 곡률, 오목한 정도가 서로 다른 품목이 존재한다. 제형별 프로토타입 하나만 사용하면 이러한 변형이 평균 과정에서 뭉개질 수 있다. 품목별 exemplar prototype을 함께 사용하는 이유가 여기에 있다.

### 5.4 이미지 전처리 의존성

배경과 알약의 색이 비슷하거나, 그림자가 알약과 연결되거나, 알약 일부가 잘린 이미지에서는 윤곽 추출이 부정확할 수 있다. 복잡한 실사진에서는 알약을 먼저 크롭하거나 별도의 세그멘테이션 방법을 사용하는 것이 더 안정적이다.

### 5.5 확률로 해석할 수 없는 점수

현재 거리와 상대 점수는 후보 간 순위를 비교하기 위한 값이며, 통계적으로 보정된 정답 확률이 아니다. 상대 점수가 높다는 이유만으로 해당 제형이 확실하다고 판단해서는 안 된다.

### 5.6 일반화 성능 평가 미완료

현재까지의 결과만으로 전체 정확도나 운영 성능을 주장할 수 없다. 동일 품목의 앞면과 뒷면 또는 유사 이미지가 학습과 평가에 동시에 포함되면 성능이 과대평가될 수 있으므로, 품목 단위로 분리한 검증이 필요하다.

## 6. 적절한 활용 방식

현재 시스템은 최종 제형을 무조건 확정하는 자동 분류기보다 다음과 같은 보조 판정 도구로 활용하는 것이 적절하다.

```text
입력 이미지
  → 형상 임베딩 검색
  → Top-k 제형 후보 축소
  → 거리와 후보 간 차이 확인
  → 고신뢰 사례만 자동 처리
  → 애매한 사례는 사용자 확인
```

권장 운영 방식은 다음과 같다.

- Top-1과 Top-2의 거리 차이가 충분히 큰 경우에만 자동 판정
- 거리가 학습 분포에서 벗어나면 판정 보류
- 경계가 유사한 제형은 Top-k 후보로 제공
- 색상, 크기, 각인 정보 등 다른 특징과 결합
- 사용자 확인 결과를 후속 데이터 개선에 활용

## 7. 후속 개선 방향

1. 품목 단위 학습·검증 분리와 leave-one-item-out 평가를 수행한다.
2. 클래스별 Top-1, Top-3 정확도와 혼동행렬을 산출한다.
3. 제형별 데이터 수에 따른 성능 차이를 분석한다.
4. 거리 및 Top-1/Top-2 차이에 대한 판정 보류 임계값을 설정한다.
5. 윤곽 특징에 CSV의 장축·단축 크기와 색상 정보를 결합한다.
6. 배경이 복잡한 이미지에 대한 알약 세그멘테이션을 개선한다.
7. 필요하면 사전학습 이미지 임베딩과 수작업 형상 특징을 결합한 하이브리드 방식을 비교한다.

## 8. 종합 평가

본 PoC는 소수 표본만으로 29개 제형을 완전 자동 분류하는 데에는 한계가 있음을 확인하는 동시에, 형상 임베딩을 이용해 가능한 제형을 Top-k로 축소하는 보조 판정 방식의 실현 가능성을 검증했다.

따라서 본 결과를 실패한 분류 모델로 평가하기보다는 다음과 같이 정리하는 것이 적절하다.

> 제한된 소수 표본 환경에서 수작업 형상 임베딩과 프로토타입 거리 기반 검색의 적용 가능성을 검증하였다. 일부 제형에서는 유효한 후보 검색 결과를 확인했으나, 형태가 유사하거나 클래스 내부 변이가 큰 제형에서는 단일 자동 판정에 한계가 있었다. 현재 결과는 최종 분류기보다는 Top-k 후보 축소와 사용자 확인을 지원하는 보조 검색 모델로 활용하는 것이 적절하다.

이번 단계의 핵심 성과는 성능을 완전히 입증한 것이 아니라, 자동화 가능한 범위와 한계를 구체적으로 확인하고 데이터 확충, 판정 보류, 다중 정보 결합이라는 후속 개선 방향을 도출한 것이다.
