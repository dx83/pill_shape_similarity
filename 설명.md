# 형상 특징 차원수와 가중치 설명

`shape_features.py`는 알약 윤곽을 총 56개 숫자로 표현한 다음, 네 특징 그룹의 중요도가 균형 있게 반영되도록 가중 유클리드 거리를 계산한다.

## 1. 56차원 구성

| 그룹 | 차원 계산 | 차원 | 의미 |
|---|---:|---:|---|
| 기본 기하 | 9개 지표 | 9 | 길쭉함, 원형성, 볼록성, 대칭성 등 |
| EFD | 12 harmonics × 2 특이값 | 24 | 전체 윤곽의 세부 형태 |
| 방사형 FFT | 1~16 주파수 | 16 | 중심에서 외곽까지 거리 변화 |
| Hu moment | 7개 모멘트 | 7 | 전체적인 형상 패턴 |
| 합계 | 9+24+16+7 | **56** | |

### 기본 기하 특징 9차원

1. `log_aspect_ratio`: 긴 변/짧은 변 비율
2. `circularity`: 원에 얼마나 가까운지
3. `eccentricity`: 타원처럼 길쭉한 정도
4. `rectangularity`: 최소 외접 사각형을 얼마나 채우는지
5. `solidity`: 볼록 껍질 내부를 얼마나 채우는지
6. `convexity`: 윤곽이 얼마나 볼록하고 매끄러운지
7. `concavity`: 오목하게 들어간 정도, `1 - solidity`
8. `symmetry_best_axis`: 두 축 중 대칭도가 높은 축
9. `symmetry_other_axis`: 나머지 축의 대칭도

`solidity`와 `concavity`는 서로 반대값이므로 오목한 형태, 특히 땅콩형이나 나비형의 특징이 기본 기하 그룹 안에서 비교적 강하게 반영된다.

### EFD 24차원

윤곽선을 12개의 주파수 성분으로 분해한다. 각 harmonic에서 2×2 계수 행렬의 특이값 2개를 사용하므로 다음과 같이 24차원이 된다.

```text
12 harmonics × 2 values = 24차원
```

낮은 harmonic은 전체적인 타원·삼각·사각 형태를, 높은 harmonic은 모서리와 오목한 부분 같은 세부 윤곽을 표현한다.

### 방사형 FFT 16차원

알약 중심에서 외곽까지의 거리를 64개 각도 구간에서 측정한 뒤 FFT를 적용한다. 평균 크기인 0번 주파수는 제외하고 1~16번 주파수만 사용한다.

따라서 크기 자체보다는 다음과 같은 반복적인 윤곽 변화를 표현한다.

- 원형처럼 반지름 변화가 적은 형태
- 타원처럼 양쪽으로 길쭉한 형태
- 땅콩처럼 중간이 들어간 형태
- 별형이나 다각형처럼 주기적인 굴곡이 있는 형태

### Hu moment 7차원

전체 윤곽의 모멘트에서 계산되는 전통적인 7개 형상 특징이다. 값의 범위가 매우 크기 때문에 로그 변환하며, 반전에 따른 부호 차이는 제거한다.

## 2. 그룹 가중치 의미

기본값은 다음과 같다.

```python
{
    "basic": 0.40,
    "efd": 0.35,
    "radial": 0.20,
    "hu": 0.05,
}
```

합계는 1.0이다.

- 기본 기하: **40%**
- EFD: **35%**
- 방사형: **20%**
- Hu moment: **5%**

이 값은 각 차원에 그대로 곱하지 않는다. 그룹의 차원수가 많다고 자동으로 영향력이 커지는 것을 막기 위해 다음처럼 차원별 가중치를 계산한다.

```text
차원별 가중치 = sqrt(그룹 가중치 / 그룹 차원수)
```

실제 값은 다음과 같다.

| 그룹 | 계산 | 차원별 가중치 |
|---|---:|---:|
| 기본 기하 | √(0.40/9) | 0.2108 |
| EFD | √(0.35/24) | 0.1208 |
| 방사형 | √(0.20/16) | 0.1118 |
| Hu | √(0.05/7) | 0.0845 |

## 3. 제곱근을 사용하는 이유

최종 비교가 유클리드 거리이기 때문이다.

```text
distance = sqrt(Σ(weighted_query - weighted_reference)²)
```

가중치를 곱한 뒤 다시 제곱하므로, 차원별로 `sqrt(weight / 차원수)`를 곱해야 그룹 전체의 제곱거리 영향력이 `weight`가 된다.

최종 제곱거리는 사실상 다음과 같다.

```text
distance² =
    0.40 × 기본 특징의 평균 제곱 차이
  + 0.35 × EFD의 평균 제곱 차이
  + 0.20 × 방사형 특징의 평균 제곱 차이
  + 0.05 × Hu moment의 평균 제곱 차이
```

따라서 EFD가 24차원이라고 해서 9차원인 기본 기하보다 자동으로 2.67배 중요해지지 않는다.

예를 들어 네 그룹의 평균 표준화 제곱 차이가 모두 1이라면 다음과 같이 계산된다.

```text
distance² = 0.40 + 0.35 + 0.20 + 0.05 = 1.0
distance = 1.0
```

## 4. 가중치 적용 전 표준화

각 특징은 단위와 값의 범위가 다르므로 먼저 다음과 같이 표준화한다.

```text
표준화값 = (원시 특징값 - 학습 데이터 중앙값) / IQR
```

그 후 값이 지나치게 커지는 것을 방지하기 위해 `-10~10`으로 제한하고, 차원별 가중치를 곱한다.

```python
standardized = clip((vector - median) / IQR, -10, 10)
result = standardized * dimension_weights
```

전체 처리 순서는 다음과 같다.

```text
56차원 추출
→ 중앙값/IQR 표준화
→ 이상값 -10~10 제한
→ 그룹별 차원 가중치 적용
→ 유클리드 거리 계산
```

## 5. 현재 가중치의 해석

이 가중치는 학습으로 자동 산출된 값이 아니라 수동으로 설정한 기본값이다.

- `basic 0.40`: 가로세로비나 오목도처럼 안정적이고 해석하기 쉬운 특징을 가장 중요하게 본다.
- `EFD 0.35`: 기본 특징으로 구분하기 어려운 세부 윤곽을 강하게 반영한다.
- `radial 0.20`: 중심 기준 굴곡 정보를 보완적으로 사용한다.
- `Hu 0.05`: 유용하지만 마스크 노이즈에 민감하고 다른 특징과 중복되는 부분이 있어 낮게 설정했다.

여기서 “기본 기하 40%”는 모든 검색 결과에서 정확히 거리의 40%를 차지한다는 뜻이 아니다. 네 그룹의 평균 표준화 오차가 비슷할 때 제곱거리의 약 40%를 담당하도록 설계했다는 의미다.
